# Agent Execution Rules & Constraints

## 1. Architectural Guardrails
- **Folder Isolation:** NEVER mix codebase files between `/landing` and `/app`. They must remain completely independent architectures.
- **No Placeholders:** Do not write mock data fields labeled "Sample" or "Lorem Ipsum". Use realistic contexts matching luxury Indian weddings (e.g., "Mridula & Rohan - Taj Lands End Concept").
- **No Hidden API Costs:** Do not implement arbitrary AI processing, background automated tagging, or heavy computation layers without explicit user permission.

## 2. Design System & UI Constraints
- **Color Palette:**
  - Backgrounds: Dark charcoal (#121212) or deep obsidian (#0A0A0A)
  - Accents: Premium emerald green (#0F5257 or #0B6623)
  - Text: High-contrast pure white (#FFFFFF) and muted silver gray (#A0AEC0)
- **Typography:** Editorial serif headers paired with ultra-clean, lightweight sans-serif body text.
- **Spacing:** Generous margins, clean alignments, zero tight borders. Everything must feel like an elite digital magazine.

## 3. Workflow Protocol
- **Incremental Code Changes:** Do not overwrite whole files if a micro-fix is requested.
- **Explain Before Action:** Briefly explain *what* structural change is about to be executed before modifying files.
